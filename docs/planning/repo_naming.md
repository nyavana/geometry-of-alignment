# Repository Naming Brainstorm

This document captures the naming options considered for the GitHub repository
of this project, kept for future reference in case we want to revisit the name.

## Final Decision

**`geometry-of-alignment`** — selected for portfolio use. It mirrors the paper
title, uses standard ML safety vocabulary, and reads as both rigorous and
accessible.

## Project Context Considered

- Title: *The Geometry of Alignment: How Safety Lives and Dies in Modern LLMs*
- Course: Columbia EECS 6699 (Mathematics of Deep Learning)
- Themes: mechanistic interpretability of refusal, abliteration as a rank-1
  perturbation, over-refusal vs genuine safety, weight-diff analysis of
  censored vs uncensored models.
- Audience target: balanced — recruiters, ML researchers, and the general
  tech community.

## Candidate Names by Style

### Academic / Research-Forward

| Name | Notes |
|---|---|
| `geometry-of-alignment` | Mirrors paper title; clear and scholarly. **Selected.** |
| `alignment-geometry` | Shorter variant; reads like a research-area name. |
| `refusal-geometry` | More specific to the refusal-direction finding. |
| `mechanistic-alignment` | Anchors the work in mechanistic interpretability. |

### Catchy / Intriguing

| Name | Notes |
|---|---|
| `where-safety-lives` | Evocative; raises a question. From the paper subtitle. |
| `fragile-guardrails` | Communicates the core finding immediately. |
| `one-rank-away` | Provocative reference to the rank-1 result. |
| `the-refusal-direction` | Sounds like a thriller; technically precise. |

### Minimal / Clean

| Name | Notes |
|---|---|
| `abliterate` | One word; the central technique. |
| `refusal-space` | Two words; describes what is mapped. |
| `safety-rank` | Double meaning around the rank-1 subspace. |
| `aligngeo` | Compact portmanteau; less polished. |

### Descriptive / Self-Documenting

| Name | Notes |
|---|---|

| `llm-alignment-mechanistic-analysis` | Fully descriptive but long. |
| `llm-refusal-analysis` | Simple and direct. |
| `safety-alignment-geometry` | Balanced length and concept coverage. |

### Hybrid / Portfolio-Optimized

| Name | Notes |
|---|---|
| `geometry-of-alignment` | Best all-rounder. |
| `fragile-guardrails` | Strong portfolio hook for product/applied roles. |
| `llm-safety-geometry` | Safe SEO bet. |

## Rationale for the Final Pick

`geometry-of-alignment` was chosen because:

1. **Coherence with paper**: matches the paper title exactly, so resume → paper
   → repo all line up.
2. **Distinctive but recognizable**: "geometry" + "alignment" together is
   unusual enough to stand out, but uses real ML safety vocabulary that
   researchers recognize.
3. **Recruiter-safe**: even without alignment context, it sounds rigorous.
4. **Ages well**: not tied to a single technique (`abliterate`) or a
   conversational hook (`fragile-guardrails`) that may feel dated later.

## Rejected (and Why)

- `abliterate` — too jargon-y and narrow; suggests only one technique.
- `aligngeo` — feels like a scratch project, not portfolio-grade.
- `llm-alignment-mechanistic-analysis` — too long and awkward to say.
